# Status Kontrak Backend vs Frontend

Dokumen ini merangkum endpoint yang **sudah ada di backend tetapi dipakai frontend dengan format yang salah**, serta endpoint yang **memang belum ada di backend**. Tujuannya supaya frontend tidak salah pakai dan tim bisa langsung tahu bagian mana yang perlu disesuaikan.

## 2. Memang Belum Ada di Backend

| Endpoint                         | Status    | Catatan                                                        |
| -------------------------------- | --------- | -------------------------------------------------------------- |
| `GET /profile`                   | belum ada | Belum ada route dan controller untuk profil.                   |
| `PUT /profile`                   | belum ada | Belum ada route dan controller untuk update profil.            |
| `PUT /users/profile`             | belum ada | Jalur alternatif ini juga belum tersedia.                      |
| `GET /kos/current`               | belum ada | Belum ada endpoint khusus untuk kos aktif saat ini.            |
| `POST /payment-methods`          | belum ada | Belum ada resource untuk metode pembayaran bank atau e-wallet. |
| `GET /payment-methods`           | belum ada | Belum ada listing metode pembayaran.                           |
| `PUT /payment-methods/{id}`      | belum ada | Belum ada update metode pembayaran.                            |

| `GET /chats/{threadId}`          | belum ada | Belum ada detail thread chat.                                  |

| `GET /residents`                 | belum ada | Belum ada endpoint daftar penghuni.                            |

## 3. Ringkasan Praktis untuk Frontend

1. Kalau endpoint ada di bagian 1, frontend boleh pakai, tetapi payload dan parsing response harus disesuaikan dengan backend sekarang.
2. Kalau endpoint ada di bagian 2, frontend jangan memanggilnya dulu karena backend memang belum menyediakan.
3. Untuk area yang paling rawan mismatch, prioritas cek adalah `register/login`, `rooms`, `payments`, `missions`, `daily-login`, dan `mytupai`.

## 4. Catatan Implementasi Backend Saat Ini

- Backend sudah mengembalikan response JSON dengan wrapper `success` di banyak endpoint, tetapi isi `data` dan nama field tidak selalu sama dengan yang ditulis frontend.
- Beberapa endpoint di checklist frontend sebenarnya sudah ada, tetapi format body yang dikirim frontend masih memakai contract lama.
- Endpoint `profile`, `payment-methods`, `chats`, dan `residents` masih perlu diimplementasikan di backend jika memang ingin dipakai frontend.
